# Campus CRM Demo

This is a tiny CRM-themed project.

Students can understand the whole project in a few minutes:

- `index.html` contains the page structure.
- `style.css` controls the visual design.
- `script.js` stores contacts, renders the list, adds new contacts, and filters by status.

## How to Run

Open `index.html` in a browser.

No install step is needed.

## Suggested Classroom Demo

Start by showing the GitHub repository and opening the three main files. Then ask Codex to make one small, reviewable change.

Example prompt:

```text
This is a beginner-friendly CRM demo.

Please add a simple "Next follow-up date" field to each contact.

Requirements:
- Keep the code easy for beginners to read.
- Do not add a framework or build tools.
- Update the form so users can enter the follow-up date.
- Show the date on each contact card.
- Explain which files you changed.
```

## Good Follow-up Tasks

- Add a search box for contact names.
- Add a delete button for each contact.
- Add a priority field such as Low, Medium, and High.
- Count how many contacts are in each status.
- Save contacts in `localStorage` so they remain after refresh.
